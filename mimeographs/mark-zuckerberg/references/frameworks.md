# Frameworks of Mark Zuckerberg

Zuckerberg relies on specific operational and strategic frameworks to manage a massive global organization while attempting to maintain the agility of a startup. These frameworks emphasize decentralized testing, shared context, and statistical approaches to complex social problems.

## The Hacker Way (Continuous Iteration)

An approach to building that involves continuous improvement, rapid prototyping, and an open meritocracy. Release small iterations quickly rather than trying to get everything right at once. Prototype ideas immediately instead of debating them for days. Foster a culture where the best idea and implementation win, regardless of hierarchy.

> "The Hacker Way is an approach to building that involves continuous improvement and iteration."
(sources: src_062)

## Small Group Context Sharing

A management framework for maintaining agility in a large organization without relying on strict hierarchical reporting. Identify a core group of 25-30 key leaders across the company. Spend significant time together going through all the context of everything happening across the company. Use this highly connected, high-trust group as a "core army" to execute cross-functional priorities.

> "I invest a lot of time in like making sure that there's a group that's very connected to each other and that have all the context on what's going on inside the company and that's sort of like the core army that can go get like anything done."
(sources: src_021)

## Precision vs. Recall Moderation Tuning

A statistical framework for configuring AI classifiers to detect and remove harmful content at scale while protecting innocent users. Evaluate the trade-off between recall (catching the highest percentage of bad content) and precision (ensuring what is caught is actually bad). Err on the side of higher confidence/precision to minimize the number of innocent accounts mistakenly deactivated, even if it means missing some harmful content.

> "changing the content filters to have to require higher confidence and precision is actually going to be the thing that reduces the vast majority of the censorship mistakes that we make."
(sources: src_024, src_023)

## Decentralized Product Testing

A system allowing any engineer to test new ideas and learn what users respond to without corporate red tape. Allow any engineer to change code and create a new branch of the product. Ship the new version to a small, controlled portion of the community. Gather quantitative and qualitative feedback. If it improves metrics, roll it into the trunk version for everyone.

> "So we build this whole framework that allows people within the company, any engineer, to change some code, create a new branch of what Facebook is, and ship that to some number of people... And if their version is an improvement, then we roll that in"
(sources: src_025)

## Progressive Model Integration (AI)

A method for developing new capabilities for foundation models by hacking interim solutions first. Build application-specific code and hand-engineer features around the current base model to unlock immediate use cases. Hack around with these interim solutions to hone intuition, then train those capabilities directly into the next generation of the base model.

> "You do a bunch of hand coding and that makes the products better, if only for the interim. That helps show the way then of what we want to build into the next version of the model."
(sources: src_033)
